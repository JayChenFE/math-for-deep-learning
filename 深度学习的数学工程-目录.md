## 第24章　优化算法：从 SGD 到 AdamW 🆕

> _本章是原目录缺失的关键模块——理解了优化器，才理解 `Trainer` 类里那些参数在干什么。_

24.1　SGD 回顾：`θ -= lr * ∇L(θ)`（纯梯度下降）

24.2　**SGD + Momentum**：给梯度加"惯性"——小球滚下山

- 物理直觉：下山的小球有动量，不会在每个坑里停下来
- 代码：对比纯 SGD 和 Momentum 在狭长峡谷函数（f(x,y)=x²+10y²）上的收敛轨迹

  24.3　**Adam**：自适应学习率

- 直觉 1（动量项）：稀疏参数大步走，活跃参数小步走
- 直觉 2（自适应项）：梯度方差大的方向走小步（谨慎），方差小的方向走大步（加速）
- **Adam 是训练 Transformer 的事实标准** —— 不理解它就无法 debug 训练曲线

  24.4　**AdamW**：把 Weight Decay 从 L2 正则化中"解耦"

- Weight Decay = 每次更新前先让参数缩小一点点：`θ *= (1 - lr * λ)`
- 与 L2 正则化在 SGD 下等价，在 Adam 下不等价 → AdamW 修复了这个 bug
- HuggingFace Trainer 默认用的就是 AdamW

  24.5　**学习率调度**：为什么 Transformer 训练前几步要小学习率？

- Warmup：从 0 线性增长到目标 lr（前 4000 步）——防止训练初期不稳定
- Cosine Decay：从目标 lr 余弦衰减到 0——平滑收束
- 代码：三种调度策略（constant、step、cosine）的 loss 曲线对比

  24.6　**Gradient Accumulation**：batch_size 不够大时的"攒批技巧"

- 代码：每 4 步才调一次 `optimizer.step()`，等效 batch_size 扩大 4 倍

## 第29章　Transformer 架构：逐块拆解 🆕（原第27章拆分）

> _读完本章，你将能逐行阅读 "Attention Is All You Need" 论文的公式。_

29.1　**输入嵌入 + 位置编码**：向量相加（第 4 章）

- Token Embedding：查表，把 token id 变成向量
- Positional Encoding：给每个位置打上"位置标签"
- 三种方案对比：Sinusoidal（原始论文）vs Learned（BERT）vs **RoPE**（LLaMA / 旋转位置编码的直觉）

  29.2　**QKV 投影**：`Q = X @ W_Q` 等（第 6 章矩阵乘法）

- 为什么要投影三次？Q、K、V 各自扮演什么角色？
- 直觉：Q 是"我要找什么"，K 是"我有什么"，V 是"找到后取出什么"

  29.3　**缩放点积注意力**：`softmax(Q @ Kᵀ / √d_k) @ V`

- Q@Kᵀ：计算所有 token 两两之间的相似度（第 5 章点积）
- √d_k：防止点积过大导致 softmax 进入饱和区（第 22 章数值稳定）
- softmax：把相似度变成概率权重（第 1.3 节）
- @V：用权重对 Value 做加权求和 = "从相关位置提取信息"

  29.4　**Causal Mask**：上三角填 `-inf`，确保 t 时刻看不到 t+1 时刻的 token

- 代码：一行 `torch.triu` 构建 mask

  29.5　**Multi-Head Attention**：拆成 h 个头各自算，最后 concat 再过线性层

- 直觉：不同头关注不同关系（语法/语义/指代/距离）
- 代码：reshape + transpose 的实现细节（头维度变换是最容易写错的地方）

  29.6　**Feed-Forward Network**：两层 Linear + GeLU

- 位置：每个 Attention 后面跟一个 FFN
- 直觉：Attention 负责"从哪取信息"，FFN 负责"处理取到的信息"
- 维度变化：d_model → 4×d_model → d_model

  29.7　**LayerNorm + 残差连接**（第 21 章 + 第 4 章）

- `x = LayerNorm(x + Sublayer(x))`
- 残差：高速公路上的一条匝道——让梯度能直达底层
- Pre-Norm vs Post-Norm：为什么现在的模型都用 Pre-Norm（先归一化再进子层）
