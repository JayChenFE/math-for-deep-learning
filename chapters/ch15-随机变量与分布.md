> 本章目标：理解随机变量（Random Variable）的本质——它不是"不确定的值"，而是"值由概率决定的变量"。掌握四种核心分布（离散/正态/均匀/伯努利）、极大似然估计（MLE）的直觉，以及四种采样策略（greedy/temperature/top-k/top-p）——这些是理解损失函数和文本生成的基础。

### 15.1　离散随机变量 —— 掷骰子与概率质量函数

现实世界充满不确定性：掷骰子结果是 1 到 6 中的哪一个？明天会不会下雨？用户会不会点击这个按钮？概率论用**随机变量**来建模这种不确定性——它不是一个确定的数值，而是一个"取值由概率决定的变量"。

对于离散情况（取值可以一个一个列出来），我们用**概率质量函数（Probability Mass Function, PMF）**来描述：P(X = x) 表示随机变量 X 恰好取值 x 的概率。所有可能取值的概率之和必须等于 1，每个概率都 ≥ 0。

最经典的离散分布例子是公平骰子——每个面出现的概率都是 1/6。直觉告诉你掷 600 次大约有 100 次 1、100 次 2……但"大约"到什么程度？我们直接模拟。

💻 **代码　掷骰子模拟：从小样本到大样本的收敛**

```python
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)

fig, axes = plt.subplots(1, 3, figsize=(15, 4))

for i, n in enumerate([30, 300, 3000]):
    rolls = np.random.randint(1, 7, size=n)
    values, counts = np.unique(rolls, return_counts=True)
    # 补齐缺失的面（np.unique 只返回出现过的值）
    full_counts = np.array([np.sum(rolls == k) for k in range(1, 7)])
    probs = full_counts / n

    axes[i].bar(range(1, 7), probs, color='steelblue', edgecolor='white')
    axes[i].axhline(y=1/6, color='red', linestyle='--', linewidth=2, label='理论概率 1/6')
    axes[i].set_title(f'n={n} 次'); axes[i].set_xlabel('点数')
    axes[i].set_ylabel('频率'); axes[i].set_ylim(0, 0.4)
    axes[i].legend(fontsize=8)

plt.suptitle('掷骰子频率随试验次数收敛到 1/6', fontsize=13)
plt.tight_layout(); plt.show()
print("收敛规律：n 越大，柱高越接近红线（大数定律——见第 18 章）")
```

📐 **定义　离散随机变量（Discrete Random Variable）**：取值集合 {x₁, x₂, ...} 有限或可数，每个取值 xᵢ 对应概率 pᵢ = P(X = xᵢ)，满足 ∑pᵢ = 1 且 pᵢ ≥ 0。

🔗 **AI 连接**：分类任务的输出（"猫" 0.7、"狗" 0.3）就是一个离散概率分布——softmax 输出的正是这样一个 PMF。第 29 章中，Transformer 每一步预测下一个 token 时，输出的 50000 维向量就是一个巨大的离散分布。

### 15.2　连续随机变量 —— 正态分布与概率密度函数

如果随机变量可以在一个区间内取任意值（如身高 165.3cm、温度 23.7°C），它就是一个**连续随机变量**。此时 P(X = 精确值) = 0（取到 165.30000... 的概率无限小），所以我们改用**概率密度函数（Probability Density Function, PDF）**：曲线下某段区间的面积 = X 落在该区间的概率。

正态分布（Normal / Gaussian）是自然界和机器学习中最常见的连续分布——测量误差、BatchNorm 层的激活值、权重初始化的随机噪声，都常用正态分布建模。它由两个参数决定：
- **μ（均值）**：分布的中心，"钟形曲线"最高点位置
- **σ（标准差）**：分布的宽度，σ 越大曲线越扁越宽

**"68-95-99.7" 规则**：约 68% 的数据落在 μ±1σ，约 95% 落在 μ±2σ，约 99.7% 落在 μ±3σ。这条规则是异常检测和数据清洗的重要直觉——偏离 3σ 以上的点可以被视为"异常"。

💻 **代码　正态分布采样与 PDF 叠加**

```python
import numpy as np
import matplotlib.pyplot as plt
from math import erf

def normal_pdf(x, mu=0, sigma=1):
    """正态分布概率密度函数"""
    z = (x - mu) / sigma
    return np.exp(-0.5 * z**2) / (sigma * np.sqrt(2 * np.pi))

def normal_cdf(x):
    """标准正态累积分布函数（用误差函数 erf 实现）"""
    return 0.5 * (1 + erf(x / np.sqrt(2)))

np.random.seed(42)

mu, sigma = 0.0, 1.0   # 标准正态
samples = np.random.randn(10000) * sigma + mu

# 直方图（密度模式：总面积为 1）
plt.figure(figsize=(10, 5))
plt.hist(samples, bins=50, density=True, alpha=0.6, color='steelblue',
         edgecolor='white', label='采样直方图')

# 叠加理论 PDF 曲线
x = np.linspace(-4, 4, 200)
pdf = normal_pdf(x, mu, sigma)
plt.plot(x, pdf, 'r-', linewidth=2.5, label=f'N({mu}, {sigma}²) 理论 PDF')

# 标注 1σ / 2σ / 3σ 区域
for k, color in zip([1, 2, 3], ['orange', 'green', 'purple']):
    plt.axvspan(mu - k*sigma, mu + k*sigma, alpha=0.08, color=color,
                label=f'±{k}σ ({normal_cdf(k)-normal_cdf(-k):.1%})')

plt.xlabel('x'); plt.ylabel('概率密度'); plt.title('标准正态分布 N(0, 1)')
plt.legend(fontsize=8); plt.show()

# 验证 68-95-99.7 规则
for k in [1, 2, 3]:
    in_range = np.mean(np.abs(samples) < k)
    expected = normal_cdf(k) - normal_cdf(-k)
    print(f"±{k}σ 内: 采样 {in_range:.3f}  |  理论 {expected:.3f}")
```

📐 **定义　连续随机变量（Continuous Random Variable）**：取值于连续区间，由 PDF f(x) 描述，满足 f(x) ≥ 0 且 ∫f(x)dx = 1。P(a ≤ X ≤ b) = ∫ₐᵇ f(x) dx = 曲线下面积。

🔗 **AI 连接**：正态分布是整个 AI 训练体系的"默认噪声模型"。权重初始化（第 25 章）用截断正态来打破对称性；BatchNorm（第 21 章）假设激活值近似正态分布来归一化；VAE（变分自编码器）的隐变量也假设服从正态分布。理解正态分布，是理解"为什么这些设计能工作"的前提。

### 15.3　均匀分布 —— 等可能的连续版本

📐 **定义　均匀分布（Uniform Distribution）**：在区间 [a, b] 上，PDF 为常数 f(x) = 1/(b−a)；区间外为 0。`X ~ Uniform(a, b)` 表示 "X 在 [a, b] 内每个位置等可能"。

均匀分布是"最公平"的分布——没有任何偏好。在机器学习中它最经典的用途是**权重初始化**：在训练开始时，我们对每个参数应该取什么值一无所知，所以用均匀分布（或截断正态）来随机赋初值。PyTorch 的 `nn.Linear` 默认就用 `kaiming_uniform_` 来初始化权重矩阵。

💻 **代码　均匀分布与权重初始化的关系**

```python
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)

# 模拟 Xavier 初始化：Uniform[-limit, limit]  where limit = sqrt(6/(fan_in + fan_out))
fan_in, fan_out = 784, 256
limit = np.sqrt(6 / (fan_in + fan_out))
weights = np.random.uniform(-limit, limit, size=(fan_in, fan_out))

fig, axes = plt.subplots(1, 2, figsize=(12, 4))

# 左：直方图
axes[0].hist(weights.flatten(), bins=50, density=True, color='steelblue', edgecolor='white')
axes[0].axhline(y=1/(2*limit), color='red', linestyle='--', linewidth=2,
                label=f'理论 PDF = 1/(2×{limit:.4f})')
axes[0].set_xlabel('w'); axes[0].set_ylabel('概率密度')
axes[0].set_title(f'Xavier Uniform 初始化\nfan_in={fan_in}, fan_out={fan_out}')
axes[0].legend()

# 右：验证均值接近 0，方差接近理论值
theoretical_var = (2*limit)**2 / 12  # Uniform[-L, L] 方差 = (2L)²/12
axes[1].bar(['理论方差', '实际方差'], [theoretical_var, weights.var()],
            color=['lightgray', 'steelblue'], edgecolor='white')
axes[1].set_ylabel('方差')
axes[1].set_title(f'方差对比 (理论 ≈ {theoretical_var:.6f})')

plt.tight_layout(); plt.show()
print(f"权重范围: [{weights.min():.4f}, {weights.max():.4f}]")
print(f"权重均值: {weights.mean():.6f} (理想=0)")
print(f"权重标准差: {weights.std():.6f}")
```

🔗 **AI 连接**：权重初始化（第 25 章）直接依赖均匀分布和正态分布。初始化的好坏决定了训练能否成功起步——分布选错，100 层网络的激活值要么爆炸要么消失。

### 15.4　伯努利分布 —— 二选一的数学表达

📐 **定义　伯努利分布（Bernoulli Distribution）**：只有 0（失败）和 1（成功）两种结果的离散分布。P(X=1) = p，P(X=0) = 1−p。记作 X ~ Bernoulli(p)。

伯努利分布虽然简单，却是许多复杂模型的基石：
- **二分类**：逻辑回归的输出就是 P(Y=1|X)，本质上是条件伯努利分布
- **Dropout**：训练时每个神经元以概率 p 被"丢弃"（乘以 0），以 1−p 保留——这是伯努利的随机变量在起作用
- **点击率预估**：广告是否被点击 → Bernoulli(p_click)
- **抛硬币、A/B 测试、用户转化率**：统统可以建模为伯努利试验

💻 **代码　伯努利试验与二项分布的联系**

```python
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)

p = 0.3  # 成功率 30%
n_trials = 1000
samples = np.random.random(n_trials) < p  # 伯努利采样：True→1, False→0
samples_int = samples.astype(int)

# n 次独立伯努利之和 = 二项分布 Binomial(n, p)
n_groups = 50  # 每组 20 次伯努利
group_successes = samples_int[:n_groups * 20].reshape(n_groups, 20).sum(axis=1)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
axes[0].bar(['失败 (0)', '成功 (1)'],
            [np.mean(samples_int == 0), np.mean(samples_int == 1)],
            color=['lightcoral', 'steelblue'], edgecolor='white')
axes[0].set_ylabel('频率')
axes[0].set_title(f'Bernoulli(p={p}) — {n_trials} 次试验')
for i, v in enumerate([1-p, p]):
    axes[0].text(i, v, f'{v:.2f}', ha='center', va='bottom', fontweight='bold')

axes[1].hist(group_successes, bins=range(0, 22), density=True, alpha=0.7,
             color='steelblue', edgecolor='white')
axes[1].set_xlabel('20 次中成功的次数'); axes[1].set_ylabel('频率')
axes[1].set_title(f'20 次伯努利之和 ~ Binomial(20, {p})')

plt.tight_layout(); plt.show()
print(f"样本均值 = {samples_int.mean():.3f} (理论 p = {p})")
print(f"样本方差 = {samples_int.var():.3f} (理论 p(1-p) = {p*(1-p):.3f})")
```

### 15.5　极大似然估计（MLE）—— 让观测数据"最可能"出现

📐 **定义　极大似然估计（Maximum Likelihood Estimation, MLE）**：给定观测数据 {x₁, x₂, ..., xₙ}，选择使这些数据出现概率最大的参数 θ。即 θ̂ = argmax P(data | θ)。

直觉：你看到一个硬币掷了 10 次，7 次正面。你认为这枚硬币的正面概率 p 是多少？答案不是 0.5——你会猜 p ≈ 0.7，因为这使得"7 正 3 反"这个结果最可能发生。这就是 MLE 的核心直觉。

对于正态分布 N(μ, σ²)，MLE 给出了优雅的闭式解：
- μ̂ = (1/n) ∑xᵢ —— 即样本均值
- σ̂² = (1/n) ∑(xᵢ − μ̂)² —— 即样本方差（注意分母是 n，不是 n−1）

在深度学习中，**交叉熵损失本质上就是在做极大似然估计**——第 19 章将展示这个深刻的等价性。理解 MLE 的直觉，你就理解了为什么损失函数被设计成那样的形式。

💻 **代码　MLE 演示：用越来越多的数据估计正态分布均值，观察收敛**

```python
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)

true_mu = 5.0
true_sigma = 2.0
n_total = 200

# 生成全部数据
data = np.random.randn(n_total) * true_sigma + true_mu

# 逐点累积：观察 μ̂ 如何随样本量增加而收敛到 true_mu
cumulative_means = np.cumsum(data) / np.arange(1, n_total + 1)

fig, axes = plt.subplots(1, 2, figsize=(13, 4.5))

# 左：μ̂ 收敛曲线
axes[0].axhline(y=true_mu, color='red', linestyle='--', linewidth=2, label=f'真实 μ = {true_mu}')
axes[0].plot(range(1, n_total + 1), cumulative_means, color='steelblue', linewidth=1.5)
axes[0].fill_between(range(1, n_total + 1),
                     true_mu - true_sigma/np.sqrt(np.arange(1, n_total + 1)),
                     true_mu + true_sigma/np.sqrt(np.arange(1, n_total + 1)),
                     alpha=0.15, color='steelblue', label='理论 ±σ/√n')
axes[0].set_xlabel('样本量 n'); axes[0].set_ylabel('μ̂ (估计均值)')
axes[0].set_title('MLE 估计量随样本增大收敛到真值')
axes[0].legend(); axes[0].set_xlim(0, n_total)

# 右：n=5 vs n=200 的似然函数对比
for n, color in [(5, 'coral'), (200, 'steelblue')]:
    sub_data = data[:n]
    mu_grid = np.linspace(true_mu - 2, true_mu + 2, 200)

    # 对数似然（忽略常数项）：log L(μ) ∝ -0.5 Σ(xᵢ−μ)² / σ²
    log_lik = -0.5 * np.sum((sub_data[:, None] - mu_grid[None, :])**2, axis=0) / true_sigma**2
    # 归一化以便可视化
    log_lik = log_lik - log_lik.max()  # 峰值对齐到 0

    axes[1].plot(mu_grid, log_lik, color=color, linewidth=2,
                 label=f'n={n}, μ̂={sub_data.mean():.3f}')
    axes[1].axvline(x=sub_data.mean(), color=color, linestyle='--', alpha=0.5)

axes[1].axvline(x=true_mu, color='red', linestyle='-', alpha=0.6, label=f'真实 μ={true_mu}')
axes[1].set_xlabel('μ'); axes[1].set_ylabel('对数似然 (归一化)')
axes[1].set_title('似然函数随样本量增大而变尖锐')
axes[1].legend()

plt.tight_layout(); plt.show()

print(f"真实 μ = {true_mu}")
print(f"n=5   时 MLE: μ̂ = {data[:5].mean():.3f}")
print(f"n=20  时 MLE: μ̂ = {data[:20].mean():.3f}")
print(f"n=200 时 MLE: μ̂ = {data.mean():.3f}")
```

MLE 的威力在于：**不需要知道真实参数，仅凭观测数据就能逼近它**。数据越多，估计越精确。逻辑回归的权重、神经网络的输出概率、甚至 GPT 预测的下一个 token——本质上都是在做 MLE（或它的正则化变体）。

🔗 **AI 连接**：第 19.5 节将揭示：**最大化似然 ⇔ 最小化交叉熵损失**。你见过的每一个 `nn.CrossEntropyLoss()`、每一个 `model.fit()`，底层都在执行 MLE。理解了这个连接，损失函数就不再是"随便选的公式"，而是有明确概率含义的必然选择。

### 15.6　采样策略 —— 从概率分布到具体选择

有了概率分布后，怎么从中"选一个"？这个问题看似 trivial——按概率随机抽不就行了吗？——但在 AI 文本生成中，**采样策略直接决定了生成质量**。同一个模型、同一个 prompt，用不同的采样策略可能产生"妙语连珠"和"胡言乱语"两种截然不同的结果。

#### 15.6.1　Greedy（贪心）：永远选概率最高的

最简单粗暴的策略：`argmax(p)`。对于掷骰子，永远选概率最高的面（但公平骰子每面都是 1/6，greedy 无意义）；对于语言模型，永远选概率最高的下一个 token。

**问题**：Greedy 采样导致文本**循环重复**——因为每次选同样的高概率 token，陷入"最安全但最无聊"的循环。试试用 GPT-2 做 greedy decoding，你会看到" 我不 知道 我不 知道 我不 知道..." 这样的灾难。

#### 15.6.2　Temperature（温度）：控制"确定性 vs 多样性"

对 softmax 前的 logits 除以温度 T：
- **T → 0**：退化为 greedy（所有概率集中到最大项）
- **T = 1**：标准 softmax，概率保留原始比例
- **T → ∞**：趋近于均匀分布（完全随机）

T < 1 让高概率项更突出（更保守、更确定），T > 1 让概率更平滑（更多样、更"有想象力"）。实践中 temperature 是最常用的调参旋钮——调到 0.7~0.9 是文本生成的常见"甜点区间"。

#### 15.6.3　Top-k 采样：只在"前 k 名"里选

将概率排序后只保留前 k 个，其余置零后重新归一化，再从中采样。k=50 是常见默认值。

**优点**：避免给极低概率的"垃圾 token"任何被选中的机会。**缺点**：k 是固定的——对于"分布很集中"的情况（只有 3 个合理选项），k=50 太宽松；对于"分布很分散"的情况（有 200 个合理选项），k=50 又太严格。这引出了更优雅的方案：

#### 15.6.4　Top-p（核采样 / Nucleus Sampling）：动态 k

不固定 k，而是从概率最高的 token 开始往下加，直到累积概率 ≥ p（例如 p=0.9）。这意味着"集中分布时只保留少数选项，分散分布时保留更多选项"——p 值自适应地决定"候选集"大小。Top-p = 0.9 是当前 LLM 的工业标准默认值。

💻 **代码　四种采样策略在同一分布上的对比**

```python
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)

# 模拟一个语言模型输出的概率分布（10 个 token 的假想 logits）
logits = np.array([3.0, 2.0, 1.5, 1.0, 0.5, 0.2, 0.1, 0.05, 0.02, 0.01])
# 这代表 token A 最可能(3.0), token J 最不可能(0.01)

def softmax(x, T=1.0):
    """稳定版 softmax，支持温度"""
    x = np.array(x, dtype=np.float64) / T  # 温度缩放
    x_max = x.max()
    e_x = np.exp(x - x_max)
    return e_x / e_x.sum()

def sample_from_probs(probs, n_samples=1000):
    """按给定概率分布采样 n 次，返回每个 token 的频率"""
    return np.random.choice(len(probs), size=n_samples, p=probs)

# 原始 softmax 概率
orig_probs = softmax(logits, T=1.0)

# 四种采样策略
strategies = {
    'Greedy': None,  # 永远选 argmax
    f'Temperature\nT=0.5': None,
    f'Temperature\nT=2.0': None,
    'Top-k\nk=3': None,
    'Top-p\np=0.9': None,
}

# --- Greedy ---
greedy_idx = np.argmax(orig_probs)
strategies['Greedy'] = np.zeros(10)
strategies['Greedy'][greedy_idx] = 1.0

# --- Temperature ---
strategies[f'Temperature\nT=0.5'] = softmax(logits, T=0.5)
strategies[f'Temperature\nT=2.0'] = softmax(logits, T=2.0)

# --- Top-k (k=3) ---
top_k = 3
topk_probs = orig_probs.copy()
topk_probs[np.argsort(topk_probs)[:-top_k]] = 0  # 只保留最大的 3 个
topk_probs /= topk_probs.sum()
strategies['Top-k\nk=3'] = topk_probs

# --- Top-p (p=0.9) ---
p = 0.9
sorted_indices = np.argsort(orig_probs)[::-1]
cumsum = np.cumsum(orig_probs[sorted_indices])
cutoff_idx = np.searchsorted(cumsum, p) + 1
topp_probs = orig_probs.copy()
topp_probs[sorted_indices[cutoff_idx:]] = 0
topp_probs /= topp_probs.sum()
strategies['Top-p\np=0.9'] = topp_probs

# ===== 可视化 =====
fig, axes = plt.subplots(2, 3, figsize=(15, 8))
axes = axes.flatten()
labels = [f'Token_{chr(65+i)}' for i in range(10)]
colors = plt.cm.viridis(np.linspace(0.15, 0.85, 10))

# 原始分布
axes[0].bar(labels, orig_probs, color=colors, edgecolor='white')
axes[0].set_title('原始分布 (T=1.0 softmax)', fontweight='bold')
axes[0].set_ylabel('概率'); axes[0].tick_params(axis='x', rotation=45)

# 五种策略
for ax, (name, probs) in zip(axes[1:], strategies.items()):
    ax.bar(labels, probs, color=colors, edgecolor='white')
    ax.set_title(name, fontweight='bold')
    ax.tick_params(axis='x', rotation=45)
    # 标注最大概率
    max_idx = np.argmax(probs)
    ax.text(max_idx, probs[max_idx], f'{probs[max_idx]:.2f}',
            ha='center', va='bottom', fontsize=8, fontweight='bold')

fig.suptitle('同一组 logits 在不同采样策略下的概率分布', fontsize=14, y=1.01)
plt.tight_layout(); plt.show()

# ===== 实际采样 1000 次对比频率 =====
print("\n--- 各策略采样 1000 次的频率分布 ---")
for name, probs in strategies.items():
    if 'Greedy' in name:
        freq = strategies['Greedy']  # 不需要采样
    else:
        samples = sample_from_probs(probs, n_samples=1000)
        freq = np.bincount(samples, minlength=10) / 1000
    top3 = np.argsort(freq)[-3:][::-1]
    top3_str = ', '.join([f'{labels[i]}={freq[i]:.1%}' for i in top3])
    print(f"  {name:18s}: Top-3 → {top3_str}")
```

关键洞察：**没有一种采样策略在所有场景下都是最优的**。Temperature 控制"创造力"的高低，top-k 和 top-p 划定"候选范围"的大小。组合使用 temperature + top-p 是当前 LLM（包括 GPT-4 和 Claude）的默认行为。

🔗 **AI 连接**：第 30.3 节将用 GPT-2 实战这四种策略——同样的 prompt，greedy 循环说废话、高 temperature 胡言乱语、top-p=0.9 产生流畅文本。采样策略是"从概率到文本"的最后一步，也是用户最能直观感受到的参数。
