## 第14章　计算图与自动微分 —— 反向传播的工程本质

> 本章目标：理解计算图（节点=操作，边=数据流），前向和后向传播的对应关系，用 PyTorch 的 `.backward()` 和 `torch.no_grad()` 掌控梯度流动。这是理解整个 PyTorch 训练循环的基石。
> 前置知识：第 13 章（链式法则）、第 6 章（矩阵乘法）

---

### 14.1~14.2　计算图与前向传播

📐 **定义　计算图（Computational Graph）**：一个有向无环图（DAG），节点是操作（+, ×, sin, matmul...），边是数据（张量）。前向传播从左到右计算结果。

例：y = (x+2)×3 → `x → [+2] → u → [×3] → y`

---

### 14.3　反向传播——从输出往回传梯度

每个节点只需要知道：(1) 输出对输入的本地梯度，(2) 从上游传来的梯度。两者相乘传给下游。

手工计算：y=(x+2)×3, ∂y/∂x = 3·1 = 3。

---

### 14.4　用 PyTorch 验证

💻 **代码　第一个 autograd 实验**

```python
import torch

x = torch.tensor(1.0, requires_grad=True)
u = x + 2      # 加节点
y = u * 3      # 乘节点

y.backward()   # 反向传播！
print(f"y = {y.item()}, ∂y/∂x = {x.grad.item()}")  # 9, 3 ✓

# 更复杂的例子
x2 = torch.tensor(2.0, requires_grad=True)
y2 = torch.sin(x2**2)
y2.backward()
print(f"sin(x²) at x=2: grad={x2.grad.item():.4f} (cos(4)*4={torch.cos(torch.tensor(4.))*4:.4f})")
```

---

### 14.6　切断梯度：detach() 与 no_grad()

📐 **`.detach()`**：创建张量副本，断开与原计算图的连接（用于分析模型输出但不影响训练）。
📐 **`torch.no_grad()`**：上下文管理器，整个代码块不构建计算图（用于推理、评估）。

💻 **代码　有/无 no_grad 的性能对比**

```python
import torch
import time

x = torch.randn(1000, 1000)

# 有梯度追踪
t0 = time.perf_counter()
for _ in range(500):
    y = x @ x.T; y.sum()
t1 = time.perf_counter()

# no_grad：不建图
with torch.no_grad():
    t2 = time.perf_counter()
    for _ in range(500):
        y = x @ x.T; y.sum()
    t3 = time.perf_counter()

print(f"有梯度: {t1-t0:.3f}s  no_grad: {t3-t2:.3f}s  加速: {(t1-t0)/(t3-t2):.1f}x")
print("推理时必须用 no_grad —— 省显存、省时间")
```
