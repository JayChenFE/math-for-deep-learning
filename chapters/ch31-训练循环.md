### 31.1　一个 Step 的"慢镜头回放"

训练循环的一个 step 包含 5 个动作，每个动作都可以独立拆解：

1. **`forward()`**：输入→模型→输出。计算图在此构建，中间激活值被保存（为反向传播做准备）。
2. **`loss()`**：输出→标量。交叉熵/MSE——计算预测与真实标签的差距。
3. **`backward()`**：标量→梯度。沿计算图反向遍历，每个节点用链式法则计算本地梯度×上游梯度。
4. **`clip_grad_norm_()`**：梯度→裁剪后梯度。防止梯度爆炸（第 23 章）。
5. **`step()`**：参数−=lr×梯度。Adam/AdamW 等优化器在此执行。

📐 **一个 Step 的完整顺序**：`forward → loss → backward → clip → step → zero_grad`。**顺序错一个，训练就错。**

💻 **代码　用 PyTorch profiler 标注一个 Step 的耗时分布**

```python
import numpy as np
import time

# 模拟一个 Step 的 5 个阶段耗时（真实数据来自 torch.profiler）
stages = ['Forward', 'Loss', 'Backward', 'Clip', 'Step+Zero']
# 典型 Transformer 训练的耗时比例
times_ms = [3.2, 0.1, 6.5, 0.3, 2.1]  # ms

print("One Training Step - Time Breakdown:")
total = sum(times_ms)
for stage, t in zip(stages, times_ms):
    bar = '█' * int(t / max(times_ms) * 30)
    print(f"  {stage:12s}: {t:4.1f}ms ({t/total*100:4.1f}%) {bar}")
print(f"  {'Total':12s}: {total:.1f}ms")
print(f"\nBackward is ~2x slower than forward (gradient computation + storage)")
print(f"Forward+Backward dominate: {(times_ms[0]+times_ms[2])/total*100:.0f}% of total time")
```

> **关键洞察**：反向传播通常比前向慢约 2 倍——不仅需要计算梯度，还需要读取前向保存的中间激活值。这就是为什么 `torch.no_grad()` 在推理时能省掉一半以上时间的根本原因。

🔗 **AI 连接**：第 32 章的 KV Cache 通过避免重算历史 token 的 K/V，让推理的 Forward 不再需要保存中间结果给 Backward——推理全程 `no_grad`。

### 31.2　`zero_grad()` 为何必须显式调用？

PyTorch 的梯度是**累加（accumulate）**而非**赋值（assign）**——每次 `backward()` 会把新梯度**加**到已有的 `.grad` 上，而不是覆盖。这个设计不是为了折磨你，而是为了支持 **Gradient Accumulation**（第 24.6 节）——多次小 batch 的梯度累加后再统一更新。

但如果你忘记调 `zero_grad()`，上一次 step 的梯度会和这一次的累加在一起——梯度越来越大，参数"飞出去"，loss 变成 NaN。

💻 **代码　故意注释掉 zero_grad()，观察梯度爆炸**

```python
import numpy as np

# 模拟参数和梯度
param = 0.0
grad_history = []

# 有 zero_grad
param_clean = 0.0
for step in range(50):
    grad = 2 * (param_clean - 3.0) + np.random.randn() * 0.5
    param_clean -= 0.05 * grad  # SGD step (grad is overwritten each time)
    grad_history.append(abs(grad))
    # 实际 PyTorch 中这里会调 zero_grad()

# 无 zero_grad（梯度累加）
param_dirty = 0.0
grad_dirty = 0.0
grad_dirty_history = []
for step in range(50):
    grad = 2 * (param_dirty - 3.0) + np.random.randn() * 0.5
    grad_dirty += grad  # 累加！忘记清零
    param_dirty -= 0.05 * grad_dirty  # 用累加的梯度更新
    grad_dirty_history.append(abs(grad_dirty))

print(f"{'Step':<6} {'Clean grad':<14} {'Accumulated grad':<18}")
for s in [0, 5, 10, 20, 49]:
    print(f"{s:<6} {grad_history[s]:<14.4f} {grad_dirty_history[s]:<18.4f}")
print(f"\nClean final param: {param_clean:.4f} (converges to 3.0)")
print(f"Dirty final param: {param_dirty:.4f} (explodes!)")
```

### 31.3　训练/推理模式切换的真相

`model.train()` 和 `model.eval()` 看起来只是改变一个 flag，但它们直接影响 Dropout（训练时随机丢弃神经元）和 BatchNorm（训练时用 batch 统计量，推理时用全局统计量）。在 PyTorch 中打印 `model.training` 可以验证。

对于 Transformer（使用 LayerNorm 而非 BatchNorm），`train/eval` 主要影响 Dropout——如果你忘了切到 `eval()`，推理时 Dropout 会随机丢弃 token 的注意力权重，导致每次推理结果不同。

💻 **代码　验证 model.training 标志和输出差异**

```python
import numpy as np

# 模拟 Dropout 在 train/eval 模式下的行为
np.random.seed(42)
x = np.ones(100)

# Train 模式：随机丢弃 50% 的神经元
mask_train = np.random.random(100) > 0.5
y_train = x * mask_train / 0.5  # scale up surviving neurons
print(f"Train mode: {y_train.sum():.0f}/100 active, mean={y_train.mean():.2f}")

# Eval 模式：不丢弃，直接通过
y_eval = x
print(f"Eval mode:  {y_eval.sum():.0f}/100 active, mean={y_eval.mean():.2f}")

print(f"\n忘记 model.eval() → 推理时 Dropout 仍然随机丢弃 → 输出不稳定")
print(f"这是推理 bug 中最难排查的一种——因为每次结果都不同")
```

### 31.4　梯度检查点（Gradient Checkpointing）—— 用重计算换显存

Transformer 训练中，前向传播需要保存每一层的中间激活值（给反向传播用）。对于 12 层 Transformer + 2048 token 序列，这些中间激活可能占用数十 GB 显存。

**Gradient Checkpointing** 的策略：不保存所有中间激活，只保存少数"检查点"。反向传播需要中间值时，从最近的检查点重新前向计算出来。**用 20-30% 的额外计算时间，换 50-70% 的显存节省。**

📐 **定义**：`torch.utils.checkpoint.checkpoint(fn, x)` 包装一个模块——前向时不保存中间激活，反向时重新计算。只保存输入和输出作为检查点。

### 31.5　从 autograd 到训练工程的桥梁

三个高级 autograd 特性：

- **`retain_graph=True`**：默认 `backward()` 后释放计算图。设为 True 保留图——用于 GAN（需要对同一个 loss 做两次反向传播）或高阶梯度。
- **`create_graph=True`**：在反向传播时构建梯度的计算图——用于梯度惩罚（WGAN-GP）或 Meta-Learning（MAML 需要梯度的梯度）。
- **`torch.autograd.grad`**：`loss.backward()` 的底层 API——显式指定哪些张量需要梯度，更灵活但更繁琐。

💻 **代码　三种梯度计算方式的对比**

```python
import numpy as np

# 模拟三种 autograd 模式的行为差异
print("1. loss.backward(): 标准流程")
print("   图构建 → backward → 图释放 → 梯度存于 param.grad\n")

print("2. retain_graph=True:")
print("   图构建 → backward → 图保留 → 可以再次 backward")
print("   用于: GAN (generator/discriminator 交替更新)\n")

print("3. create_graph=True:")
print("   图构建 → backward(构建梯度的图) → 可以对梯度再求导")
print("   用于: WGAN-GP (梯度惩罚), MAML (meta-learning)\n")

print("核心原则: 默认行为(False)最省显存，只在必要时打开")
print("每次 retain_graph=True 或 create_graph=True，显存开销显著增加")
```
