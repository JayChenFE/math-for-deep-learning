## 第5章　向量的点积（内积） —— 相似度的度量

> 本章目标：掌握点积的两种等价定义——代数（对应相乘再求和）和几何（|a||b|cosθ）——并理解它们为什么是同一件事。用余弦相似度构建一个"猜你喜欢"推荐系统，并建立全书最重要的 AI 连接：**Transformer 的 Q·K 注意力得分就是点积。**
> 前置知识：第 3 章（向量、shape）、第 4 章（向量加减数乘）

---

### 5.1　代数定义 —— 对应位置相乘再求和

两个向量 "像不像"？一个最朴素的想法是：把它们的对应分量乘起来，再加总。如果两个向量在每一维上都"同高同低"，乘积就会又大又正；如果"你高我低"，乘积就会抵消变小。

这就是点积（Dot Product）的代数定义——简单到只需要一行代码，但它是一切"相似度"计算的起点。

📐 **定义　点积 / 内积（Dot Product / Inner Product）**：对两个 n 维向量 **a** = [a₁, a₂, ..., aₙ] 和 **b** = [b₁, b₂, ..., bₙ]，点积 **a**·**b** = a₁b₁ + a₂b₂ + ... + aₙbₙ = Σ aᵢbᵢ。结果是一个**标量**（单个数字），不是向量。

💻 **代码　三种方式计算点积，验证它们完全等价**

```python
import numpy as np

a = np.array([2.0, 3.0, 1.0])
b = np.array([4.0, 1.0, 5.0])

# 方式 1: 手工循环——理解"对应相乘再求和"的含义
dot_manual = sum(a[i] * b[i] for i in range(len(a)))

# 方式 2: np.dot —— NumPy 标准做法
dot_np = np.dot(a, b)

# 方式 3: @ 运算符 —— PyTorch 风格（Python 3.5+）
dot_at = a @ b

print(f"手工循环: 2×4 + 3×1 + 1×5 = {dot_manual}")
print(f"np.dot:   {dot_np}")
print(f"a @ b:    {dot_at}")
print(f"三者相等: {dot_manual == dot_np == dot_at} ✓")

# 点积的符号揭示了方向关系
a_pos = np.array([3.0, 4.0])
b_pos = np.array([2.0, 1.0])   # 与 a 大致同向
b_neg = np.array([-2.0, -1.0])  # 与 a 大致反向（夹角 > 90°，点积为负）
b_orth = np.array([-4.0, 3.0])  # 与 a 垂直（3×(−4)+4×3=0）——等下会验证

print(f"\na·b_pos  = {a_pos @ b_pos:.1f}  (正值 → 夹角 < 90°)")
print(f"a·b_neg  = {a_pos @ b_neg:.1f}  (负值 → 夹角 > 90°)")
print(f"a·b_orth = {a_pos @ b_orth:.1f}  (零 → 垂直，夹角 = 90°)")
```

> **关键洞察**：点积的正负号藏着两个向量的"方向关系"——点积 > 0 意味着它们大致指向同一侧（夹角小于 90°），点积 < 0 意味着它们大致指向两侧（夹角大于 90°），点积 = 0 意味着它们恰好垂直。这个简单的事实将在 5.2 节被几何公式 `|a||b|cosθ` 完美解释。

🔗 **AI 连接**：在 PyTorch 中，`F.linear(input, weight)` 的内部实现就是 `input @ weight.T + bias`——矩阵乘法的每一个元素都是一次点积。第 29 章 Transformer 的 Q @ Kᵀ 中，`(batch, heads, seq, d_k) @ (batch, heads, d_k, seq)` 的每一对 (query_i, key_j) 之间算的就是一个 d_k 维向量的点积。

### 5.2　几何本质 —— 点积 = 投影长度 × cosθ

代数定义 `Σ aᵢbᵢ` 好用但缺乏直觉——"把一堆数乘起来再加总"怎么就变成"相似度"了？几何定义回答了这个"为什么"：

$$**a**·**b** = ‖**a**‖ ‖**b**‖ \cos\theta$$

翻译成人话：**点积 = a 的长度 × b 的长度 × 两向量夹角的余弦。** 如果夹角 θ=0°（完全同向），cos 0° = 1，点积取最大值 ‖a‖‖b‖。如果 θ=90°（垂直），cos 90° = 0，点积 = 0。如果 θ=180°（完全反向），cos 180° = −1，点积取最小值 −‖a‖‖b‖。

从这个公式可以自然导出**余弦相似度（Cosine Similarity）**——把长度因素剥掉，只保留"方向一致性"：

$$\cos\_sim( **a**, **b**) = \frac{**a**·**b**}{‖**a**‖ ‖**b**‖} = \cos\theta$$

余弦相似度永远在 [−1, 1] 之间：1 = 完全同向，0 = 垂直（不相关），−1 = 完全反向。**它是推荐系统和语义搜索的标配度量**——不关心向量的"绝对大小"（一篇文章写 100 字还是 10000 字），只关心"方向"（文章在聊什么话题）。

💻 **代码　亲手验证：代数定义和几何定义给出完全相同的答案**

```python
import numpy as np

def cosine_similarity(a, b):
    """余弦相似度——度量两个向量指向方向的接近程度"""
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

np.random.seed(42)
a = np.random.randn(3)
b = np.random.randn(3)

# 方式 1: 代数——对应相乘再求和
dot_algebraic = np.dot(a, b)

# 方式 2: 几何——|a||b|cosθ
# 先用余弦定理算 cosθ
cos_theta = np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
# 反推点积
dot_geometric = np.linalg.norm(a) * np.linalg.norm(b) * cos_theta

print(f"代数定义  a·b = {dot_algebraic:.6f}")
print(f"几何定义 |a||b|cosθ = {dot_geometric:.6f}")
print(f"两者相等: {abs(dot_algebraic - dot_geometric) < 1e-10} ✓")
print(f"\ncosθ = {cos_theta:.4f}")
print(f"夹角 θ = {np.arccos(cos_theta) * 180 / np.pi:.1f}°")
print(f"余弦相似度 = {cosine_similarity(a, b):.4f}")

# ===== 极端情况验证 =====
print("\n--- 极端情况 ---")
v = np.array([3.0, 4.0])
same = v.copy()                          # 完全同向
opposite = -v                            # 完全反向
orthogonal = np.array([-4.0, 3.0])       # 垂直（点积应为 0）

print(f"v·v (同向):     {v @ same:.1f}  → cos= {cosine_similarity(v, same):.1f}")
print(f"v·(-v) (反向):  {v @ opposite:.1f}  → cos= {cosine_similarity(v, opposite):.1f}")
print(f"v·[-4,3] (垂直): {v @ orthogonal:.1f}  → cos= {cosine_similarity(v, orthogonal):.1f}")
# 验证：v 和 [-4,3] 确实垂直——3×(-4)+4×3 = -12+12 = 0 ✓
```

> **关键洞察**：两个公式 `Σ aᵢbᵢ` 和 `|a||b|cosθ` 给出完全相同的数字——这就是线性代数中"代数"和"几何"的交汇点。你可以在纸上用代数算点积（不需要画图），然后立刻知道两向量夹角的余弦（不需要量角器）。Transformer 的 Q @ Kᵀ 正是利用了这一点：在 64 维空间中，**你没法"画出"两个向量，但你可以用点积精确知道它们的夹角——从而知道"query 想要什么"和"key 有什么"之间的匹配度。**

🔗 **AI 连接**：余弦相似度是推荐系统、语义搜索、人脸验证、文本去重等 AI 应用的基础度量。第 9 章 LoRA 中的低秩分解也依赖于向量空间中的方向度量。Transformer 第 29 章——**整个自注意力机制就是 "对 Q 和 K 的所有组合求缩放后的点积，再 softmax 变成权重"**，而那个"点积"就是你此刻彻底搞懂的这个操作。

### 5.3　"猜你喜欢" —— 用点积构建第一个推荐系统

你现在有了构建推荐系统的全部数学工具。思路出奇简单：

1. 将每个用户表示为一个向量（从他们的历史行为中抽取：喜欢的文章类别、点击过的商品属性等）
2. 将每个待推荐物品也表示为一个向量
3. 计算用户向量与每个物品向量的**余弦相似度**（或直接算点积）
4. 返回相似度最高的 Top-N 个物品

这就是**基于内容的推荐（Content-Based Recommendation）**的数学骨架。真正的工业推荐系统会加更多层（协同过滤、深度学习排序等），但数学核心始终是一个操作：**点积 → 排序 → Top-K。**

💻 **代码　用户 vs 电影推荐：用点积排序，挑出最匹配的 3 部**

```python
import numpy as np

# ===== 电影特征向量：[动作, 喜剧, 科幻, 爱情, 悬疑] =====
# 每部电影用一个 5 维向量描述各类型的"浓度"
movies = {
    "星际穿越": np.array([0.3, 0.0, 0.9, 0.2, 0.1]),
    "泰坦尼克":  np.array([0.1, 0.1, 0.0, 0.95, 0.1]),
    "盗梦空间":  np.array([0.6, 0.0, 0.7, 0.1, 0.5]),
    "喜剧之王":  np.array([0.0, 0.95, 0.0, 0.2, 0.0]),
    "黑客帝国":  np.array([0.8, 0.0, 0.6, 0.1, 0.3]),
    "爱乐之城":  np.array([0.0, 0.3, 0.0, 0.9, 0.0]),
}

# 用户偏好向量——通过用户历史行为提取（喜欢科幻和悬疑，对爱情无感）
user_profile = np.array([0.2, 0.05, 0.8, 0.05, 0.6])

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b) + 1e-8)

# 计算用户与每部电影的匹配度
print("用户偏好: 科幻 > 悬疑 > 动作 >> 爱情/喜剧\n")
print(f"{'电影':<10} {'点积':<10} {'余弦相似度':<12}")
print("-" * 32)

scores = {}
for name, movie_vec in movies.items():
    dot = np.dot(user_profile, movie_vec)
    cos_sim = cosine_similarity(user_profile, movie_vec)
    scores[name] = cos_sim
    print(f"{name:<10} {dot:<10.4f} {cos_sim:<12.4f}")

# 排序推荐 Top-3
top3 = sorted(scores.items(), key=lambda x: x[1], reverse=True)[:3]
print(f"\n🎯 推荐 Top-3:")
for i, (name, score) in enumerate(top3, 1):
    print(f"  {i}. {name} (余弦相似度: {score:.4f})")
print("\n推荐逻辑: 用户喜欢科幻+悬疑 → '盗梦空间'(科幻+悬疑)和'星际穿越'(科幻)排名最高")
```

> **关键洞察**：推荐系统的本质是**向量空间中的最近邻搜索**。用户向量和物品向量在同一个语义空间中——离得近的就是好推荐。当物品量从 6 部电影扩展到数百万商品时，暴力扫描所有点积就太慢了——工业界用**近似最近邻（ANN）**索引（如 FAISS）来加速，但底层操作的数学还是点积。

### 5.4　AI 连接：Transformer 中的 Q·K —— 注意力 = 点积的规模化应用

现在来到全书最重要的 AI 连接之一。

在 Transformer 的每一层、每一个注意力头中，都会执行这个操作：`scores = Q @ Kᵀ / sqrt(d_k)`。拆开来看：

- **Q**（Query）："我要找什么"——当前 token 的查询向量，shape = (seq_len, d_k)
- **K**（Key）："我有什么"——所有 token 的键向量，shape = (seq_len, d_k)
- **Q @ Kᵀ**：计算 query_i 与 key_j 的**点积**，结果 shape = (seq_len, seq_len)——一个"所有 token 对之间匹配度"的方阵
- **/ sqrt(d_k)**：缩放因子——防止 d_k 太大导致点积值过大，softmax 后概率全部集中在最大值（变得和 greedy 没区别）

而 `Q @ Kᵀ` 这个矩阵乘法中的每一个元素 `scores[i][j]`，就是第 i 个 query 向量和第 j 个 key 向量的**点积**——和你刚才在 5.3 节计算"用户向量·电影向量"是完全一样的数学操作，只是在 64 维空间中对 `seq_len × seq_len` 对向量同时计算。

📐 **核心类比**：Q @ Kᵀ 中的每一个点积 = "query 想找什么"和"key 有什么"之间的**余弦相似度（未归一化版）**。相似度越高 → token_j 对 token_i 的注意力权重越大 → token_j 的信息更多地被"聚合"到 token_i 的表示中。

💻 **代码　亲手实现单头缩放点积注意力——20 行纯 NumPy**

```python
import numpy as np

np.random.seed(42)

# 模拟参数
seq_len = 4          # 4 个 token 的短序列
d_k = 8              # 每个头 8 维（真实 Transformer 用 64）

# 随机初始化 Q, K, V（训练中这些是通过 X @ W_Q 等投影得到的——第 29 章）
Q = np.random.randn(seq_len, d_k)
K = np.random.randn(seq_len, d_k)
V = np.random.randn(seq_len, d_k)

# ===== 三行代码实现注意力核心 =====
# 1. 计算点积得分矩阵 (4×8) @ (8×4) = (4×4)
scores = Q @ K.T                          # 每个 scores[i][j] 就是 Q_i · K_j !

# 2. 缩放 + softmax → 注意力权重
d_k = Q.shape[-1]
attention_weights = np.exp(scores / np.sqrt(d_k))  # 稳定版省略了 max 减去（见第 22 章）
attention_weights = attention_weights / attention_weights.sum(axis=-1, keepdims=True)

# 3. 加权聚合 Value → 输出
output = attention_weights @ V           # (4×4) @ (4×8) = (4×8)

print(f"得分矩阵 (Q @ Kᵀ):\n{scores.round(2)}\n")
print(f"注意力权重 (每行之和=1):\n{attention_weights.round(3)}\n")
print(f"每行之和: {attention_weights.sum(axis=1).round(3)} ← 每个 query 对所有 key 的权重和为 1 ✓")
print(f"\n输出 shape: {output.shape} ← 每个 token 的新表示是 V 的加权和")

# ===== 核心洞察：查看 0 号 token 对谁的注意力最高 =====
token0_weights = attention_weights[0]
most_attended = np.argmax(token0_weights)
print(f"\nToken 0 最关注的 token: Token {most_attended} "
      f"(权重={token0_weights[most_attended]:.3f})")
print("这就是 'token 0 觉得 token_j 和自己最相关' 的数学表达——来自一个点积。")
```

> **关键洞察**：你刚刚用 20 行 NumPy 实现的，就是 Transformer 论文中 `Scaled Dot-Product Attention` 的完整数学逻辑。拆到最底层——`scores[i][j] = Q[i]·K[j]`——就是第 i 个查询向量和第 j 个键向量的**点积**。整个 LLM 时代建立在 5.1 节那个 `sum(a_i * b_i)` 操作之上。你现在可以自信地说：**我理解了大语言模型注意力机制的数学原子。**

### 5.5　Agent 记忆检索：点积驱动 RAG —— Query 与知识库的"相亲大会"

第 4.5 节你看到了 Agent 如何用向量加法做状态迁移。现在加上点积：**Agent 在新的状态下，如何从海量记忆中检索最相关的信息？** 答案就是 RAG（Retrieval-Augmented Generation）的核心操作——**用点积给文档和用户 Query 的"匹配度"打分。**

RAG 的工作流程出奇简单：

1. 将知识库中的每篇文档预先编码为一个向量（embedding）
2. 将用户 Query 也编码为同维度的向量
3. 计算 **Query 向量与每篇文档向量的点积（或余弦相似度）**
4. 返回相似度最高的 Top-K 篇文档作为"参考上下文"
5. LLM 基于这些上下文生成回答

你会发现，这和 5.3 节"猜你喜欢"推荐系统是**完全相同的数学骨架**——只是把"用户"换成了"Query"，把"电影"换成了"文档"。推荐和检索，在向量空间中是同一个问题：**点积 → 排序 → Top-K。**

💻 **代码　RAG 检索模拟：100 篇文档中检索最相关的 Top-5**

```python
import numpy as np

np.random.seed(42)

# ===== 模拟知识库：100 篇文档，每篇编码为 128 维向量 =====
n_docs, d_embed = 100, 128
doc_vectors = np.random.randn(n_docs, d_embed)
doc_vectors = doc_vectors / np.linalg.norm(doc_vectors, axis=1, keepdims=True)  # 归一化

# 给文档加"主题标签"（模拟真实场景中某些文档与 Query 更相关）
# 构造 5 篇"真正相关"的文档：让它们在第 0~15 维度上有一个共同的强信号
relevant_ids = [12, 37, 55, 78, 91]
for rid in relevant_ids:
    doc_vectors[rid, :16] += np.random.randn(16) * 0.8
    doc_vectors[rid] = doc_vectors[rid] / np.linalg.norm(doc_vectors[rid])

# ===== 用户 Query 向量——模拟"想找机器学习相关文档" =====
query = np.random.randn(d_embed)
query[:16] += np.random.randn(16) * 0.8  # 同样强化前 16 维（与相关文档的方向对齐）
query = query / np.linalg.norm(query)

# ===== RAG 检索：Query 与所有文档的点积 → 排序 → Top-K =====
scores = doc_vectors @ query   # (100,) —— 每个元素 = 一篇文档与 Query 的余弦相似度！
top_k = 5
top_indices = np.argsort(scores)[::-1][:top_k]

print(f"知识库: {n_docs} 篇文档 (dim={d_embed})")
print(f"Query: 强化了前 16 维的随机向量\n")
print(f"{'Rank':<6} {'Doc ID':<8} {'相似度':<10} {'是否预设相关'}")
print("-" * 42)
for rank, idx in enumerate(top_indices, 1):
    is_relevant = "✓" if idx in relevant_ids else ""
    print(f"{rank:<6} {idx:<8} {scores[idx]:<10.4f} {is_relevant}")

# 分析相似度分布
print(f"\n相似度分布:")
print(f"  Top-5 平均相似度: {scores[top_indices].mean():.4f}")
print(f"  全库平均相似度:   {scores.mean():.4f}")
print(f"  Top-5 / 全库均值:  {scores[top_indices].mean()/scores.mean():.1f}x")
print(f"\n检索命中率: {sum(1 for i in top_indices if i in relevant_ids)}/5 篇预设相关文档被召回")
```

> **关键洞察**：RAG 检索的本质就是 5.3 节推荐系统的翻版——**Query 向量是"用户"，文档向量是"商品"，点积是"匹配度"，Top-K 是"推荐列表"。** 唯一的区别是 RAG 多了一步：检索到的文档被拼接到 LLM 的 prompt 中作为"参考上下文"，让模型基于事实回答而非凭空想象。第 4.5 节的状态迁移 + 本节的点积检索 + LLM 的生成 = 一个完整的 Agent 认知循环。

🔗 **AI 连接**：工业级 RAG 系统用 FAISS 或 Milvus 做十亿级文档的近似最近邻检索，但底层数学还是点积。第 19 章的信息论会解释为什么交叉熵损失会驱动 embedding 模型学会"将语义相似的文本映射到相近的向量"——这正是 RAG 能工作的根本原因。
