### 5.1　代数定义——对应相乘再求和

📐 **定义　点积（Dot Product）**：两个同维度向量对应位置相乘，再将所有乘积相加。`**a**·**b** = Σ aᵢ·bᵢ`。结果是一个**标量**。

在 NumPy 中用 `np.dot(a, b)` 或直接 `a @ b`（`@` 是 Python 3.5+ 的矩阵乘法运算符，对一维向量就是点积）。

💻 **代码　点积的手工实现与 NumPy 验证**

```python
import numpy as np

a = np.array([2, 3, 1])
b = np.array([4, 0, 5])

# 手工：对应相乘再求和
dot_manual = sum(a[i] * b[i] for i in range(len(a)))
print(f"手工点积: {dot_manual}")  # 2*4 + 3*0 + 1*5 = 13

# NumPy 多种方式
print(f"np.dot(a, b): {np.dot(a, b)}")   # 13
print(f"a @ b:       {a @ b}")           # 13
print(f"(a * b).sum(): {(a * b).sum()}") # 13

# 验证：点积结果是一个标量
result = np.dot(a, b)
print(f"result 类型: {type(result)}, 值: {result}")
# 标量：不是数组，是一个单纯的数
```

### 5.2　几何本质——方向匹配度的标尺

点积不止是机械的乘加运算，它有一个关键的几何解释：

📐 **几何公式**：`**a**·**b** = ‖**a**‖·‖**b**‖·cos(θ)`，其中 θ 是两向量夹角。

由此导出：
- `**a**·**b** > 0` → cos(θ) > 0 → 夹角 < 90°，方向大致相同
- `**a**·**b** = 0` → cos(θ) = 0 → 夹角 = 90°，**垂直（正交）**——两向量"不相关"
- `**a**·**b** < 0` → cos(θ) < 0 → 夹角 > 90°，方向大致相反

📐 **定义　余弦相似度（Cosine Similarity）**：`cos_sim(**a**, **b**) = (**a**·**b**) / (‖**a**‖·‖**b**‖)`。值域 [−1, 1]，只衡量方向，不关心长度。1 表示完全相同方向，0 表示正交，−1 表示完全相反。

💻 **代码　验证代数与几何公式等价**

```python
import numpy as np

a = np.array([3, 0])       # 沿 x 轴
b = np.array([1.5, 1.5])   # 45° 方向

# 代数
dot_alg = np.dot(a, b)

# 几何
norm_a = np.linalg.norm(a)
norm_b = np.linalg.norm(b)
cos_theta = dot_alg / (norm_a * norm_b)
theta = np.arccos(cos_theta)  # 弧度
dot_geo = norm_a * norm_b * np.cos(theta)

print(f"代数点积: {dot_alg:.4f}")
print(f"几何点积: {dot_geo:.4f}")
print(f"夹角: {np.degrees(theta):.1f}°")
print(f"余弦相似度: {cos_theta:.4f}")  # ≈ 0.707 (cos 45°)

# 三种典型情况
pairs = [
    ([1, 0], [1, 0],   "同向 → cos=1"),
    ([1, 0], [0, 1],   "正交 → cos=0"),
    ([1, 0], [-1, 0],  "反向 → cos=-1"),
]
for v1, v2, desc in pairs:
    v1, v2 = np.array(v1, dtype=float), np.array(v2, dtype=float)
    cs = np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))
    print(f"{desc}: cos = {cs:.0f}")
```

### 5.3　代码实现"猜你喜欢"——点积推荐系统

推荐系统的一个核心思路是：把用户和物品都表示成向量，用余弦相似度（或点积）衡量它们的匹配度。

💻 **代码　简易推荐系统：找最匹配的商品**

```python
import numpy as np

# 用户偏好向量：[价格敏感度, 科技偏好, 时尚偏好, 品质要求]
user = np.array([0.2, 0.9, 0.1, 0.7])

# 5 个商品的特征向量（相同维度）
items = {
    "机械键盘": np.array([0.0, 1.0, 0.0, 0.8]),
    "运动鞋":   np.array([0.5, 0.0, 0.9, 0.3]),
    "蓝牙耳机": np.array([0.1, 0.9, 0.2, 0.6]),
    "帆布袋":   np.array([0.9, 0.0, 0.6, 0.1]),
    "显示器":   np.array([0.0, 1.0, 0.0, 0.9]),
}

def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b) + 1e-8)

# 计算每个商品的匹配度并排序
scores = {name: cosine_sim(user, vec) for name, vec in items.items()}
ranked = sorted(scores.items(), key=lambda x: x[1], reverse=True)

print("推荐排名（余弦相似度从高到低）：")
for i, (name, score) in enumerate(ranked, 1):
    bar = "█" * int(score * 30)
    print(f"  {i}. {name:6s} {score:.4f} {bar}")
```

🔗 **AI 连接**：这个排序逻辑和 Transformer 中的 Self-Attention 完全同构——你把 `user` 换成 Query **Q**，把 `items` 换成 Key **K**，把"相似度分数"换成 softmax 归一化后的注意力权重。**Q@Kᵀ 计算的就是一批查询向量和一批键向量之间的全部点积**（第 29 章）。

### 5.4　点积的向量化批处理

在实际 AI 系统中，你永远不会一条一条地计算相似度——你一次处理整个 batch。

💻 **代码　批量计算点积矩阵**

```python
import numpy as np

# 4 个用户查询向量（batch=4, d=3）
Q = np.random.randn(4, 3)
# 6 个候选物品键向量（num_items=6, d=3）
K = np.random.randn(6, 3)

# 一次性计算所有 (用户, 物品) 对的点积
scores = Q @ K.T  # shape: (4, 6)
print(f"scores shape: {scores.shape}")  # (4, 6)
print(f"scores[0] 是用户 0 对所有 6 个物品的打分: {scores[0]}")

# 每个用户选 top-2 物品
top2_idx = np.argsort(scores, axis=1)[:, -2:]  # shape: (4, 2)
print(f"\n每个用户推荐的第1和第2物品索引:\n{top2_idx}")
```

这就是 Transformer 中 `Q @ K.transpose(-2, -1)` 的原型——一个矩阵乘法，算出整个序列中每个 token 对其他所有 token 的注意力分数。你在第 5 章就已经掌握了第 29 章的核心操作。
