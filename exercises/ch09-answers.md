## 1. （概念）SVD 公式 U Σ Vᵀ 三部分各自的角色

**答案**：用"旋转→拉伸→旋转"的句式：
- **Vᵀ**（右奇异向量转置）：对输入空间做**第一次旋转**——把原始坐标轴旋转到"拉伸方向"上。
- **Σ**（奇异值对角矩阵）：沿新坐标轴**独立拉伸**每个方向——σ₁ 拉最长、σ₂ 次之……值越大该方向越重要。
- **U**（左奇异向量）：对拉伸后的结果做**第二次旋转**——转回最终输出空间的朝向。

任意矩阵对向量的作用 = 先旋转（对准拉伸方向）→ 再拉伸（各方向独立缩放）→ 再旋转（转到输出朝向）。

---

## 2. （概念）为什么取前 k 个奇异值就能近似？

**答案**：奇异值从大到小排列，σ₁ 最大，σ_n 最小（甚至接近 0）。大奇异值对应矩阵的"主要信息方向"，小奇异值对应"噪声/可忽略的细节"。前 k 个大奇异值通常占 80-99% 总能量（Σσᵢ²），丢弃后 n−k 个小奇异值损失的能量极小，重构矩阵和原始矩阵几乎一样。

k 的选择是一个**压缩率 vs 精度**的权衡：k 越小 → 存储越少 → 压缩率越高 → 但重构误差越大。通常选择能量达 90% 的最小 k 作为"甜点"。

---

## 3. （概念）LoRA 中 r=8 在 SVD 框架下意味着什么？

**答案**：r=8 意味着 LoRA 假设微调更新 ΔW 的信息集中在**前 8 个奇异值方向**上——ΔW 的"有效秩"约为 8。用 B(d×8) @ A(8×d) 来近似完整的 ΔW(d×d)。

- r 越大（如 r=64）：保留更多奇异值方向 → 更接近全参数微调效果 → 但参数量增加、训练更慢。
- r 越小（如 r=4）：更激进的压缩 → 训练更快、显存更省 → 但可能丢失重要方向、微调效果下降。

r=8~16 是实践中常用的甜点区间——用 0.1%~1% 的参数量达到接近全参数微调的效果。

## 4. （代码）SVD 低秩近似：重构误差 vs k

```python
import numpy as np

np.random.seed(42)

# 生成 100×80 随机矩阵
A = np.random.randn(100, 80)
U, S, VT = np.linalg.svd(A, full_matrices=False)

total_energy = np.sum(S**2)

print(f"{'k':<6} {'重构误差(Frobenius)':<22} {'能量保留':<12}")
print("-" * 40)

for k in [5, 10, 20, 40]:
    # 用前 k 个奇异值做低秩近似
    A_approx = U[:, :k] @ np.diag(S[:k]) @ VT[:k, :]

    # Frobenius 范数 = sqrt(sum of squared differences)
    error = np.linalg.norm(A - A_approx)  # Frobenius norm

    # 能量保留比例
    energy_kept = np.sum(S[:k]**2) / total_energy * 100

    print(f"k={k:<4}  {error:<22.4f}  {energy_kept:<11.1f}%")

# 对比原始矩阵和近似矩阵的 Frobenius 范数
print(f"\n原始矩阵 Frobenius 范数: {np.linalg.norm(A):.4f}")
print("观察：k 越大 → 误差越小 → 能量保留比例越高")
print("k=40 时（保留一半奇异值）能量已接近 80%+，重构误差显著降低")
```

**预期输出**：k=5 时误差较大（~80% 原始范数），k=40 时误差显著降低（~30%），能量保留比例从 50% 提升到 85%+。k 的选择决定了压缩率与精度的平衡点。
