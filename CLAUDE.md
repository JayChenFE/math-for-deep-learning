## 一、符号约定体系（全书锁定，禁止漂移）

### 1.1 数学符号

| 类别    | 符号                   | 示例                       | 说明                      |
| ------- | ---------------------- | -------------------------- | ------------------------- |
| 标量    | 小写斜体               | `x`, `y`, `n`              | 普通数值                  |
| 向量    | 粗体小写               | **x**, **w**, **θ**        | 在 Markdown 中用 `**x**`  |
| 矩阵    | 粗体大写               | **W**, **X**, **Q**        | 在 Markdown 中用 `**W**`  |
| 张量    | 书法体大写（或粗体）   | **𝒳** 或明确维度           | 三阶及以上，用 shape 区分 |
| 转置    | 上标 ᵀ                 | **X**ᵀ                     | 用 Unicode `ᵀ`，不用 `^T` |
| 点积    | `a·b` 或 `**a**·**b**` | 代数/几何两种形式          |
| L2 范数 | `‖**x**‖₂`             | `\|\|x\|\|` 用 Unicode `‖` |
| 偏导数  | `∂f/∂x`                | 用 Unicode `∂`，不用 `d`   |
| 梯度    | `∇f` 或 `∇**θ**L`      | 用 Unicode `∇`             |
| 期望    | `E[**X**]`             |                            |
| 概率    | `P(A                   | B)`                        | 条件概率用竖线            |
| 熵      | `H(P)`                 |                            |
| KL 散度 | `D_KL(P‖Q)`            | 双竖线                     |

### 1.2 代码中的符号对应

| 数学符号         | Python 变量名       | 示例                           |
| ---------------- | ------------------- | ------------------------------ |
| **x** (向量)     | `x`                 | `x = torch.randn(d)`           |
| **W** (矩阵)     | `W`                 | `W = torch.randn(d_in, d_out)` |
| **X** (批量输入) | `X`                 | `X.shape = (batch, d_in)`      |
| **θ** (参数)     | `params` 或单独命名 | `w, b = ...`                   |
| ∂L/∂**W** (梯度) | `W.grad`            | autograd 命名                  |

### 1.3 形状约定

- 输入形状始终标注在注释中：`# X: (batch, seq_len, d_model)`
- 向量默认为列向量，但在代码中为一维张量
- 变换后形状变化用注释说明：`# Q @ K.T: (seq_len, seq_len)`

## 二、代码规范（全书统一）

### 2.1 环境与导入模板

每章代码块依赖以下环境，首次出现时在章首声明：

```python
# 环境要求：Python 3.10+, PyTorch 2.0+, NumPy, Matplotlib
import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np
import matplotlib.pyplot as plt
import matplotlib.animation as animation
from matplotlib import cm
```

### 2.2 命名约定

- 变量：`snake_case`（`hidden_dim`, `learning_rate`, `num_heads`）
- 常量：`UPPER_SNAKE_CASE`（`EPS = 1e-8`）
- 函数：`snake_case`（`def compute_attention(Q, K, V)`）
- 类：`PascalCase`（仅 autograd 玩具引擎等场景）
- 每章代码独立可运行，不跨章依赖自定义模块

### 2.3 代码块规范

- 所有代码块必须**自包含**：一个代码块内 `import` 齐全，可直接复制运行
- 关键计算结果用 `print(f"...")` 或 `assert` 验证
- 形状敏感的运算后加注释：`# shape: (batch, d_out)`
- 禁止使用已弃用的 API（如 `torch.softmax` 的旧 `dim` 语法）
- 优先用 `torch.` 而非 `F.`，仅在明确需要无参数版本时用 `F.relu` 等

### 2.4 代码风格

```python
# 好的示例
def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q: (batch, n_heads, seq_len, d_k)
    K: (batch, n_heads, seq_len, d_k)
    V: (batch, n_heads, seq_len, d_v)
    """
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2, -1) / np.sqrt(d_k)  # (batch, heads, seq, seq)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
    attn_weights = torch.softmax(scores, dim=-1)
    return attn_weights @ V, attn_weights
```

### 3.1 每节必须包含的结构元素（缺一不可）

> ⚠️ **以下六要素是第15章的写作标准，所有章节必须对齐此标准。** 不允许出现"一句话定义 + 代码块"的裸节。

每节按此顺序组织：

```
[节首动机] → [直觉展开] → [定义块] → [代码块] → [关键洞察] → [AI 连接]
```

**① 节首动机（2-4 句）**

用具体、读者能切身感受的现实场景引入本节概念。禁止直接抛定义。

- ✅ "你掷了 10 次硬币，7 次正面。你觉得这枚硬币公平吗？"
- ✅ "身高 165.3cm、温度 23.7°C——这些测量值可以落在区间的任何位置。"
- ❌ "离散随机变量是指……"（直接抛定义）
- ❌ "本节我们学习正态分布。"（空洞的章节预告）

**② 直觉展开（一小段，3-6 句）**

在给出任何形式化定义之前，先用日常语言把概念"翻译"一遍。回答三个问题：这是什么？为什么需要它？它和其他概念的关系是什么？

- 可以用 **加粗关键词** 突出核心术语
- 可以列举 2-3 个具体应用场景
- 复杂概念（如 15.6 采样策略）应拆为子节（N.M.1, N.M.2...），每个子节独立展开

**③ 定义块**

📐 开头标注"定义"，加粗术语 + 英文原名。定义前应已有一句平实的直觉翻译，定义本身用精确的数学语言。

```
📐 **定义　术语（English Name）**：精确的一两句话定义。可附加公式。
```

**④ 代码块**

💻 开头标注"代码　[一句描述代码目的的话]"

代码要求：
- 自包含（完整 import），可直接复制运行
- 注释讲"故事"——不只写 `# 计算均值`，而是 `# n 越大，mu_hat 越接近 true_mu`
- 关键输出用 `print` 打印，输出内容有解释
- 涉及随机数时设置 `np.random.seed(42)` 或 `torch.manual_seed(42)`
- 可视化代码必须有标题（`plt.title` / `plt.suptitle`）

**⑤ 关键洞察（1-3 句，代码后）**

代码块之后，用自然语言总结读者应该从代码中获得的洞察。不是复述代码干了什么，而是解释 **代码结果意味着什么**。

- ✅ "n=5 时 μ̂ 在 3.8~6.2 之间剧烈波动，n=200 时 μ̂ 稳定在 4.9~5.1——数据越多，MLE 越可靠。"
- ❌ "以上代码演示了 MLE。"（什么都没说）

**⑥ AI 连接**

🔗 开头标注"AI 连接"，具体说明此概念在深度学习中的应用。要求：
- 引用具体的后续章节号（如"第 25 章"）
- 引用具体的 API 或函数名（如"`nn.Linear` 的 `kaiming_uniform_`"）
- 给出具体的数字（如"50000 维向量"而非"高维向量"）
- 说明"为什么这个数学概念对 AI 工程师重要"

### 3.2 字数控制（参考值，质量优先于数量）

| 指标 | 参考范围 | 说明 |
|------|---------|------|
| 每节正文（不含代码） | 250-800 字 | 简单概念可短，核心概念可长；**禁止为了凑字数注水** |
| 每章总字数（含代码） | 3000-5500 字 | 第15章约 4300 字，以此为标杆 |

> **核心原则**：字数下限是"读者能理解"的下限，不是"写完了"的下限。如果一节只有 2 句定义 + 代码，**无论字数多少都是不合格的**。

## 七、习题规范

### 7.1 题目类型比例

- 概念题（标记"概念"）：40-50%，检验理解深度
- 代码题（标记"代码"）：50-60%，检验落地能力
- 总题数：每章 3-6 题

### 7.2 题目设计要求

- 概念题：要求一句话回答，不能是"请解释 XXX"这种模糊题
  - ❌ "请解释什么是点积"
  - ✅ "两个向量点积为 0 是什么意思？点积为正/负分别代表什么？"
- 代码题：明确给出输入/预期行为的描述
  - ❌ "实现一个神经网络"
  - ✅ "对给定的 logits `[2.0, 1.0, 0.1]`，手动实现 softmax，验证所有输出 >0 且和为 1"

### 7.3 答案规范

- 概念题答案：1-3 句话，直接给出结论
- 代码题答案：给出完整可运行代码 + 预期输出注释

## 十、生成流程（逐章执行）

生成每一章时，按以下 SOP 操作：

### 输入准备

> ⚠️ **首次运行**：如果 `state/` 目录尚不存在，先创建该目录并初始化两个空 JSON 文件（`concepts-introduced.json` → `{}`，`cross-refs.json` → `[]`）。

1. 读取本章在目录中的标题和每节标题
2. 读取 `state/concepts-introduced.json`——确认本章依赖的概念已全部引入
3. 读取 `state/cross-refs.json`——确认本章是否有前序章节的引用需要兑现

### 生成
 
4. 按"章节结构模板"生成正文（直觉→公式→代码）
5. 正文中的"AI 连接"和"Transformer 定位"用 🔗 标注
6. 章末钩子必须指向目录中明确存在的后续章节

### Notebook 生成

7. 将章节正文和所有代码块转换为 `notebooks/chapter_NN.ipynb`
   - 正文段落 → Markdown 单元格（保留定义块 📐、AI 连接 🔗 等标注）
   - 代码块 → Code 单元格（保持自包含，含完整 `import`）
   - 习题 → 单独的 Markdown 单元格，提示读者新建代码单元格作答
   - 章末钩子 → 最后的 Markdown 单元格，附加"运行 Kernel → Restart & Run All 验证"提示
8. 在 notebook 的 metadata 中设置 `kernelspec` 为 Python 3，`language_info.version` 为项目要求的 Python 版本

### 校验

9. 提取所有代码块，检查 import 完整性、形状兼容性
10. 对照"术语对照表"，确保首次出现的术语标注了英文
11. 对照符号约定，确保符号使用与全书一致
12. 验证 notebook JSON 结构有效，所有 cell 的 `cell_type` 和 `source` 正确

### 习题答案生成

13. 生成 `exercises/chNN-answers.md`，包含本章全部习题的完整答案
    - 概念题：1-3 句话直接结论（符合 §7.3 规范）
    - 代码题：完整可运行代码 + 注释标注的预期输出
    - 代码答案必须**实际运行验证**，确保输出与注释一致
14. 答案文件末尾标注"答案校验通过"及校验日期

### 状态更新

15. 更新 `concepts-introduced.json`（新增本章引入的概念）
16. 更新 `cross-refs.json`（如果本章有新的前瞻引用，登记；如果兑现了旧引用，标记完成）
17. 在 `generation-log.md` 中记录生成时间、模型版本、发现的坑

## 十一、已知的坑（持续积累）

- `torch.softmax` 在 PyTorch 2.0+ 中 `dim` 参数建议显式指定，不依赖默认行为
- `np.linalg.eig` 对非对称矩阵可能返回复数特征值，教学示例用对称矩阵
- 第二章的数值微分在 h 太小时会出现舍入误差（约 h=1e-8 以下），需在正文中说明
- 第 15.6 节的采样策略代码中，top-p 的累积求和需要先对概率排序再 `cumsum`
